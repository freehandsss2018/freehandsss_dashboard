## 2. 四層根因分析 (4-Layer Root Cause Analysis)

### Layer 1: Profit Auditor 回傳格式錯誤 (CRITICAL — 假警報的直接原因)

**n8n Code Node v2 的 `runOnceForAllItems` 模式強制要求回傳 `[{json: {...}}]` 陣列格式。**

```text
❌ 舊代碼: return auditResults;           → 回傳裸物件 {auditPassed: true, ...}
✅ 修復:   return [{ json: auditResults }]; → 回傳 n8n 標準格式
```

**連鎖反應：**

- Profit Auditor 回傳裸物件 `{auditPassed: true, ...}`
- 下游 Auditor Logic Switch (Switch Node v1) 接收到的 `$json.auditPassed` 為 `undefined`
- Switch Node 的 boolean 比較將 `undefined` 視為 `false`
- → **永遠路由到 Output 0 (Auditor Alert)** → 每筆訂單都發 Telegram 警報

**重要機制解釋：**
Switch Node v1 boolean comparison:

```json
{
  "mode": "rules",
  "dataType": "boolean",
  "value1": "={{ $json.auditPassed }}",
  "rules": {"rules": [{"operation": "equal", "value2": false, "output": 0}]},
  "fallbackOutput": 1
}
```

- 當 `$json.auditPassed` = `undefined` 時，n8n 將其視為 falsy → matches `equal false` → Output 0
- Output 0 → Auditor Alert → Telegram 🚨 警報
- Output 1 (fallback) → 死端（無連接，僅為 side-channel）

### Layer 2: Parse Items SKU 正規化範圍過窄 (Cost=$0 的直接原因)

**Dashboard 前端發送的 Product_Name 格式多樣，但舊版 Parse Items 只匹配最窄的格式：**

```text
❌ 舊代碼: sku.includes("木框套裝")  → 只匹配 "木框套裝"
✅ 修復:   sku.includes("木框")      → 匹配 "木框套裝", "木框款式", "立體擺設(木框)" 等所有變體
```

**舊版做法：** 對 SKU 做 in-place 字串替換（如把 `3肢` 替換為 `4肢`），但不重寫完整 SKU
**新版做法：** 完整重寫為 Airtable Product_Database 中的 canonical SKU 格式：

- 含「木框」→ `木框套裝 (4肢)` 或 `木框套裝 (2肢)`
- 含「玻璃瓶」→ `玻璃瓶套裝 (4肢)` 或 `玻璃瓶套裝 (2肢)`

**同樣適用於玻璃瓶：**

```text
❌ 舊代碼: sku.includes("玻璃瓶套裝") → 遺漏 "玻璃瓶款式" 等變體
✅ 修復:   sku.includes("玻璃瓶")     → 匹配所有玻璃瓶變體
```

### Layer 3: 鎖匙扣/吊飾 Mode 串接順序錯誤

**舊版在木框正規化之前就串接 Mode，造成邏輯衝突。V45.7.4 確保：**

1. 先執行木框/玻璃瓶正規化
2. 再執行鎖匙扣/吊飾 Mode 串接

### Layer 4: Windows curl 中文編碼損壞 (測試期間的隱藏陷阱)

**Windows 終端 (cp950 codepage) 使用 inline curl `-d '{...中文...}'` 時，會將中文字元損壞為 Unicode 替代字元 (U+FFFD)。**

```text
❌ Inline curl:  curl -d '{"Items":[{"Product_Name":"木框套裝 (4肢)"}]}'
   → Hex: efbfbdefbfbd (U+FFFD 替代字元)
   → Airtable filterByFormula 搜尋亂碼 → 無匹配 → Cost=$0

✅ 檔案方式:
   1. Python 寫入 UTF-8 JSON 檔案
   2. curl -d @file.json -H "Content-Type: application/json; charset=utf-8"
   → Hex: e69ca8e6a186 (正確的「木框」UTF-8 編碼)
   → Airtable 正確匹配 → Cost=$210 ✅
```

**驗證方法：**

```bash
python3 -c "
import json
payload = {...}  # 含中文的 JSON
with open('test.json', 'w', encoding='utf-8') as f:
    json.dump(payload, f, ensure_ascii=False)
"
curl -X POST https://yanhei.synology.me:8443/webhook-test/fetch-fhs-order \
  -H "Content-Type: application/json; charset=utf-8" \
  -d @test.json
```

> ⚠️ **鐵律：在 Windows 上向 n8n webhook 發送含中文的 JSON，永遠不要用 inline curl。必須先寫入 UTF-8 檔案，再用 `curl -d @file.json`。**

**完整修復後代碼：**

// Profit Auditor V45.7.4 - Bible V3.7 Financial Guard ($5 Tolerance)
// FIX: return [{json:...}] format required by n8n Code v2 runOnceForAllItems
const body = $("Receive Dashboard Order").first().json.body || $("Receive Dashboard Order").first().json;

const DIFF_THRESHOLD = 5;

const FRAME_PRICES = { "2肢": 2080, "4肢": 2380 };
const BOTTLE_PRICES = { "2肢": 1380, "4肢": 1680 };
const KEYCHAIN_S_ADDON = [0, 860, 1200, 1680, 2000];
const KEYCHAIN_P_SOLO  = [0, 1580, 2160, 2940, 3520];

const JEWELRY_SOLO     = [0, 2280, 3080, 3880];

    theoreticalTotal: 0,

    customer: body.Customer_Name  || "Unknown",

            hasMainProduct = true;
            const limbKey = (name.includes("4肢") || name.includes("3肢")) ? "4肢" : "2肢";

        } else if (name.includes("玻璃瓶")) {
            hasMainProduct = true;
            const limbKey = (name.includes("4肢") || name.includes("3肢")) ? "4肢" : "2肢";
            theoreticalTotal += BOTTLE_PRICES[limbKey] || 1680;

        } else if (name.includes("鎖匙扣")) {
            const table = hasMainProduct ? KEYCHAIN_S_ADDON : KEYCHAIN_P_SOLO;

            } else {
                theoreticalTotal += table[4] + (qty - 4) * 500;

### 3.2 Parse Items & Generate SKU (Node: `97e25cdd-619b-4033-b189-8f2fbb654ac4`)

**完整修復後代碼：**

```javascript
// === V45.7.4 Parse Items & Generate SKU (SKU Normalization Layer) ===
const body = $("Receive Dashboard Order").first().json.body || $("Receive Dashboard Order").first().json;
const orderId = body.Order_ID || "未命名";
const customer = body.Customer_Name || "未命名";
const appDate = body.Appointment_Date || null;
const revenue = (body.Deposit || 0) + (body.Balance || 0) + (body.Additional_Fee || 0);
const rawText = body.Full_Order_Text || "";
const rawState = body.Raw_Form_State || "";

let outputItems = [];

// V45.7.4: Support both Order_Items_List (Dashboard) and Items (legacy/test)
const itemsList = body.Order_Items_List || body.Items;

if (itemsList && Array.isArray(itemsList)) {
    for (const item of itemsList) {
        let sku = item.Product_Name || "";
        const qty = Number(item.Quantity) || 1;

        // === 1. 3D Display SKU Normalization (Bible V3.7 Section 4) ===
        // Handles: 木框款式/木框套裝/立體擺設(木框) → 木框套裝 (N肢)
        // Rule: 3肢 maps to 4肢 pricing per V3.7
        if (sku.includes("木框")) {
            let limb = (sku.includes("4肢") || sku.includes("3肢")) ? "(4肢)" : "(2肢)";
            sku = `木框套裝 ${limb}`;
        } else if (sku.includes("玻璃瓶")) {
            let limb = (sku.includes("4肢") || sku.includes("3肢")) ? "(4肢)" : "(2肢)";
            sku = `玻璃瓶套裝 ${limb}`;
        }

        // === 2. Keychain SKU Normalization ===
        // Dashboard sends: "嬰兒鎖匙扣 - 不銹鋼" + Mode: "(加購)"
        // Airtable expects: "嬰兒鎖匙扣 - 不銹鋼 - {qty}飾 {mode}"
        if (sku.includes("鎖匙扣") && item.Mode) {
            sku = `${sku} - ${qty}飾 ${item.Mode}`;
        }

        // === 3. Jewelry/Charm SKU Normalization ===
        // Dashboard sends: "嬰兒吊飾 - 925銀" + Mode: "(加購)"
        // Airtable expects: "嬰兒吊飾 - 925銀 - {qty}飾 {mode}"
        if (sku.includes("吊飾") && item.Mode) {
            sku = `${sku} - ${qty}飾 ${item.Mode}`;
        }

        let shipping_saved = (sku.includes("鎖匙扣") && qty > 1) ? (qty - 1) * 20 : 0;
        let necklace_saved = (sku.includes("吊飾")) ? Math.floor(qty / 2) * 220 : 0;

        outputItems.push({
            json: {
                Order_ID: orderId,
                Customer_Name: customer,
                Appointment_Date: appDate,
                Total_Revenue: revenue,
                Order_Text: rawText,
                Raw_Form_State: rawState,
                Search_SKU: sku,
                Original_Qty: qty,
                Item_Notes: item.Notes || "",
                Order_Item_Key: item.Order_Item_Key || "",
                Shipping_Deduction: shipping_saved,
                Necklace_Deduction: necklace_saved
            }
        });
    }
} else {
    outputItems.push({ json: { Order_ID: orderId, Customer_Name: customer, Total_Revenue: revenue, Order_Text: rawText, Raw_Form_State: rawState, Search_SKU: "無商品", Shipping_Deduction: 0, Necklace_Deduction: 0 }});
}
return outputItems;
```

## 6. 工作流架構 (24 Nodes, Production)

```text
Receive Dashboard Order → Input Normalizer → Switch Action
  ├─ Output 0 (delete) → Search Record to Delete → Delete Record → Notify Telegram (Delete)
  └─ Output 1 (create/edit) → [PARALLEL]:
      ├─ Profit Auditor → Auditor Logic Switch → Auditor Alert (side-channel, non-blocking)
      └─ Parse Items & Generate SKU → Batch SKU Collector → Read Cache File
         → Smart Cache Strategist → Fetch Exact Base Cost → Local Data Mapper
         → Calculate Profit & Pack Items → Create Main Order → Bind Main Order ID
         → Create Sub Items → Pack Telegram Data → Send Profit Report
```

**注意事項：**

- `Cache Hit?` node 存在但已斷開連接（dead node），不影響運行
- `Smart Cache Strategist` 直接連接到 `Fetch Exact Base Cost`
- `Pack Telegram Data` 是第 24 個 node（之前的 23-node 版本缺少此 node）
- Auditor Alert 是 side-channel（非阻塞），不影響主訂單處理流程

## 8. 教訓與防護措施 (Lessons Learned)

### 🔴 教訓 1: n8n Code Node v2 回傳格式

**所有 n8n Code node (v2, runOnceForAllItems) 必須回傳 `[{json: {...}}]` 格式。**
回傳裸物件不會報錯，但下游 Switch/IF node 會收到 `undefined` 值，導致靜默失敗。

### 🔴 教訓 2: Windows 終端 + 中文 = 編碼地雷

**永遠不要在 Windows 終端使用 inline curl 發送含中文的 JSON。**
必須用 Python/Node.js 寫入 UTF-8 檔案，再用 `curl -d @file.json`。

### 🟡 教訓 3: SKU 正規化要廣不要窄

Dashboard 前端可能發送多種格式（木框套裝、木框款式、立體擺設(木框)），SKU 正規化應使用最寬鬆的匹配（`includes("木框")`），然後重寫為 Airtable 的 canonical 格式。

### 🟡 教訓 4: n8n API PUT 的欄位限制

PUT 更新工作流時必須清理：

- 移除 `active`（唯讀）
- 移除 node 上的 `issues`（非標準）
- 移除 settings 中的 `availableInMCP`、`binaryMode`（非標準）

### 🟢 教訓 5: 絕對禁止 Import From File

n8n 的 Import From File 會建立全新工作流 + 全新 Webhook URL，破壞 Dashboard 現有連線。所有修改必須在 n8n editor 內直接修改，或透過 API PUT 更新。
