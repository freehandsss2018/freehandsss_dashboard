# five — 五個為什麼根因分析

> **Master 定義**。橋接版位於 `.claude/commands/five.md`。

**技能來源**：awesome-claude-code（FHS 適配版）

---

## 用途

對任何問題、異常、錯誤連續追問「為什麼」5 次，強制找到系統性根因而非表面症狀。

**最佳搭配**：`/debug-guide`（找到根因後再開始修復）

## 執行步驟

收到 `/five [問題描述]` 後，執行以下分析：

### 格式

```
問題：[具體描述問題]

為什麼 #1：[第一層原因]
  → 因為：[具體事實/觀察，非猜測]

為什麼 #2：[深一層原因]
  → 因為：[具體事實/觀察]

為什麼 #3：[更深層原因]
  → 因為：[具體事實/觀察]

為什麼 #4：[接近根本原因]
  → 因為：[具體事實/觀察]

為什麼 #5：[根本原因]
  → 因為：[具體事實/觀察]

根因總結：[一句話描述根本原因]
修復建議：[針對根因的解決方案，非症狀處理]
預防措施：[如何防止同類問題再次發生]
```

## 規則

1. 每一個「為什麼」必須基於**可觀察的事實**，不得是猜測
2. 若在第 N 層無法找到事實支撐，必須**先收集數據**再繼續
3. 若 5 層之後仍未找到可修復的根因，表示需要更多數據或問題範圍設定有誤
4. **禁止**將「人為疏失」作為最終根因 — 疏失的背後必有系統性原因

## FHS 常見場景

| 問題 | 典型根因方向 |
|-----|------------|
| n8n Workflow 靜默失敗 | Code Node 未回傳 `[{json:{...}}]` 格式 |
| 財務數據不符 | SKU 正規化未執行 / Airtable 公式介入計算 |
| Dashboard 顯示異常 | try-catch 吞掉了 TDZ 錯誤 |
| Airtable 數據遺失 | Upsert key 設定錯誤 |

## 使用範例

```
/five 為什麼 Total_Cost 在某些訂單上顯示為 0？
```
